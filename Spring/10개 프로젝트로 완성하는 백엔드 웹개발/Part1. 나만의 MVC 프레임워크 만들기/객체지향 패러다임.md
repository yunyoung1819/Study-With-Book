## :pushpin: 객체지향 패러다임

### 1.테스트 코드 실습
자바 단위 테스팅 프레임워크
- JUnit5 사용
- AssertJ: 테스트 코드 가독성을 높여주는 자바 라이브러리

테스트 코드를 작성하는 이유?
- 문서화 역할
- 코드에 결함을 발견하기 위함
- 리팩토링 시 안정성 확보
- 테스트하기 쉬운 코드를 작성하다 보면 더 낮은 결합도를 가진 설계를 얻을 수 있음 

TDD
- Test Driven Development (테스트 주도 개발)
- 프로덕션 코드보다 테스트 코드를 먼저 작성하는 개발 방법
- TFD(Test First Development) + 리팩토링
- 기능 동작을 검증 (메소드 단위)
